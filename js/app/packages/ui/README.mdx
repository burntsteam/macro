## Guidelines for Good Components

### Do
1. Be [SOLID](https://en.wikipedia.org/wiki/SOLID)
1. Be pure functions – for the same props, they render the same results.
1. Use tailwind, and twMerge
1. Be as simple as possible and no simpler
1. Have a [story](https://storybook.js.org/docs/writing-stories/) with a [play function](https://storybook.js.org/docs/writing-stories/play-function) (vitest!)
1. Use Kobalte as a last resort

### Don't
1. [Ever have external margins](https://mxstbr.com/thoughts/margin)
1. Use inline styles

## Storybook Gotchas

### Reactivity

Because we are in solidJS land, unless things are built to be reactive, they **will not update** from controls.

If you have a custom render function in a story, you need to use the `context` arg instead of `args`:

```ts 
const meta = {
  component: MyComponent,
  
  // What storybook docs say to do *will not* react:
  render: (args) => <MyComponent {...args} />

  // Instead, use the context provided by `storybook-solidjs-vite`:
  render: (_, context) => <MyComponent {...context.args} />
}
```
